# Family Tree Web вЂ” Setup Guide

## Prerequisites

| Tool | Version | Install |
|------|---------|---------|
| Node.js | 18+ | https://nodejs.org |
| npm | 9+ | bundled with Node |
| Wrangler CLI | 3+ | `npm install -g wrangler` |
| Python | 3.8+ | https://python.org |
| openpyxl | any | `pip install openpyxl` |

Log in to Cloudflare via Wrangler before running any of the steps below:

```bash
wrangler login
```

---

## Step 1 вЂ” Install dependencies

```bash
cd family-tree-web
npm install
```

---

## Step 2 вЂ” Create a D1 database

```bash
wrangler d1 create family-tree-db
```

Copy the `database_id` from the output and paste it into `wrangler.toml`:

```toml
[[d1_databases]]
binding = "DB"
database_name = "family-tree-db"
database_id = "PASTE_YOUR_ID_HERE"   # <-- replace this
```

---

## Step 3 вЂ” Create an R2 bucket

```bash
wrangler r2 bucket create family-tree-images
```

After creation, configure a public URL for the bucket in the Cloudflare dashboard
(Workers & Pages в†’ R2 в†’ family-tree-images в†’ Settings в†’ Public Access), then update
`wrangler.toml`:

```toml
[vars]
NEXT_PUBLIC_R2_PUBLIC_URL = "https://pub-XXXX.r2.dev"   # your public bucket URL
```

---

## Step 4 вЂ” Run database migrations

Apply the schema to your **local** dev database and the **remote** production database:

```bash
npm run migrate:local
npm run migrate:remote
```

---

## Step 5 вЂ” Import data from Excel

Place your `data.xlsx` file in the **parent** directory of `family-tree-web/`
(i.e., at `../data.xlsx` relative to the project root), then run:

```bash
python3 scripts/import_from_excel.py
```

This generates `migrations/0002_import.sql`. Then load it into both databases:

```bash
npm run import:local
npm run import:remote
```

The script reads these sheets from `data.xlsx`:

| Sheet | Columns used |
|-------|-------------|
| Main | `person`, `person_full`, `spouse_full`, `parent`, `tooltip`, `spouse` |
| images | `person`, `image` |
| links | `person`, `link`, `description` |
| gdrives | `description`, `link` |

---

## Step 6 вЂ” Set the admin password secret

```bash
wrangler pages secret put ADMIN_PASSWORD
```

Enter a strong password when prompted. This is used to log in at `/admin`.

---

## Step 7 вЂ” Local development preview

```bash
npm run preview
```

This builds the Next.js app via `@cloudflare/next-on-pages` and starts a local
Wrangler dev server that emulates D1 and R2. Open http://localhost:8788.

> **Tip:** For a faster inner loop during development you can also use `next dev`
> directly (`npx next dev`), but you won't have D1/R2 bindings without Wrangler.

---

## Step 8 вЂ” Deploy to Cloudflare Pages

```bash
npm run pages:deploy
```

On first deploy, Wrangler will ask you to select or create a Pages project.
Follow the prompts, then your site will be live on a `*.pages.dev` subdomain.

To use a custom domain, add it in the Cloudflare dashboard under
Workers & Pages в†’ your project в†’ Custom domains.

## Admin workflow

1. Visit `/admin` в†’ redirected to `/admin/login` if not authenticated.
2. Enter the `ADMIN_PASSWORD` secret.
3. **Submissions tab** вЂ” review, approve, or reject public submissions.
   - Approving a "person" submission inserts into the `people` (or `spouses`) table.
   - Approving a "link" submission inserts into the `links` table.
   - Approving a "correction" marks it resolved; the admin applies changes manually in the People tab.
4. **People tab** вЂ” search, edit, or delete people directly. Add new people without going through submission flow.
5. **Links tab** вЂ” add or delete links for any person.

---

## Updating the tree after importing new data

If you update `data.xlsx` with new entries:

```bash
python3 scripts/import_from_excel.py
# Review the generated SQL, then:
npm run import:local    # test locally
npm run import:remote   # apply to production
```

The import uses `INSERT OR IGNORE`, so existing rows are not overwritten.
To update existing rows you'll need to write UPDATE statements manually or
use the Admin dashboard.
